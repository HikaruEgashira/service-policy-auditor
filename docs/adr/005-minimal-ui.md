# ADR-005: ミニマルUI設計

## Status
Accepted

## Context
拡張機能のPopup UIはスペースが限られている（400px幅）。情報を効率的に表示する必要がある。

## Decision
shadcn/ui風のグレースケール・ミニマルデザインを採用。

- 枠線を最小限に
- タブ型ナビゲーション（Services / Events）
- サービス一覧は1行表示（ドメイン + タグ）
- イベントログはモノスペースフォント
- バッジでサービス数を表示

## Consequences
- 情報密度が高く一覧性が良い
- クリックでプライバシーポリシーに直接遷移
- 拡張アイコンのバッジで状態が一目でわかる
